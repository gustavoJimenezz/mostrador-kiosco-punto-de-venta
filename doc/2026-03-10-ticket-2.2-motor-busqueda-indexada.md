# Motor de Búsqueda Indexada (5,000+ SKUs)

**Fecha:** 2026-03-10
**Ticket:** 2.2 — Epic 2: Infraestructura y Adaptadores de Datos

## Resumen

Optimización del motor de búsqueda de productos por nombre para cumplir el SLA de < 50ms con 5,000 registros. Se reemplazó la búsqueda `ILIKE %query%` (full table scan) por `MATCH ... AGAINST` en BOOLEAN MODE, aprovechando el índice FullText `ix_products_name_fulltext` ya existente en la tabla `products`. Se incorporan tests unitarios (sin DB) y de integración con fixture de 5,000 productos.

## Cambios Principales

- Refactorizado `search_by_name` en `MariadbProductRepository` con estrategia dual: FullText para queries ≥ 3 chars e ILIKE como fallback
- Agregados tres helpers privados: `_build_fulltext_query`, `_search_by_name_fulltext`, `_search_by_name_ilike`
- Creado directorio `tests/infrastructure/persistence/` con estructura de fixtures de integración
- Implementados 10 tests unitarios con sesión mockeada y 7 tests de integración (incluyendo test de latencia < 50ms)
- Registrado marker `integration` en pytest para separar tests que requieren MariaDB real
- Sin nueva migración Alembic — el índice FullText ya fue creado en `1bafd69c0714`

## Flujo de Trabajo

```
[search_by_name(query)]
        │
        ├── len(query) < 3 ──────────────────────────────────► [_search_by_name_ilike]
        │                                                               │
        └── len(query) >= 3                                             │
                │                                                       │
                ▼                                                       │
        [_build_fulltext_query]                                         │
         "coca cola" → "+coca* +cola*"                                  │
                │                                                       │
                ▼                                                       │
        [_search_by_name_fulltext]                                      │
         MATCH(name) AGAINST(... IN BOOLEAN MODE)                       │
                │                                                       │
                ├── resultados vacíos ──────────────────────────────────┤
                │   (ej: stopwords del servidor)                        │
                │                                                       ▼
                └── resultados encontrados ──────────► [Lista de hasta 50 Products]
```

**Punto de entrada:** `search_by_name(query: str)` en `MariadbProductRepository`.

**Proceso:** Si el query tiene ≥ 3 chars, se construye una expresión BOOLEAN MODE con todos los tokens como términos obligatorios (`+token*`). El `*` activa prefix matching en el índice FullText. Si FullText retorna vacío (stopwords configurados en el servidor), se reintenta con ILIKE. Para queries < 3 chars, se va directo a ILIKE ya que `ft_min_word_len = 3` impediría que FullText indexe tokens tan cortos.

**Resultado:** Lista de hasta 50 productos ordenados alfabéticamente. Queries de una sola palabra (`"coca"`) y multi-palabra (`"coca cola"`) son compatibles con el mismo método.

## Notas Técnicas

### Expresión BOOLEAN MODE

Cada token del query se convierte en un término obligatorio con wildcard de prefijo:

| Query ingresado | Expresión BOOLEAN MODE |
|-----------------|------------------------|
| `"coca"` | `+coca*` |
| `"coca cola"` | `+coca* +cola*` |
| `"  alfajor  jorgito  "` | `+alfajor* +jorgito*` |

El operador `+` hace que cada término sea obligatorio (semántica AND). El `*` habilita prefix matching dentro del índice FullText. La parametrización siempre usa `.bindparams()` (nunca f-strings) para prevenir comportamientos inesperados con inputs especiales del modo BOOLEAN.

### Aislamiento de tests con SAVEPOINT

El fixture `db_session` usa `session.begin_nested()` en lugar de truncar tablas entre tests. Cada test hace rollback al SAVEPOINT al finalizar, dejando los 5,000 productos del fixture `populated_products` intactos durante toda la sesión de pytest. Esto reduce el tiempo de setup de O(n·tests) a O(1).

### Variable de entorno para tests de integración

```bash
# Ejecutar solo tests unitarios (sin DB)
pytest tests/ -m "not integration"

# Ejecutar tests de integración (requiere MariaDB)
POS_TEST_DB_URL=mysql+pymysql://pos:pos@localhost/kiosco_test pytest -m integration -v
```

### Índice FullText en fixture de integración

El fixture `engine` crea el índice con `IF NOT EXISTS` para soportar reruns sin teardown completo:

```sql
CREATE FULLTEXT INDEX IF NOT EXISTS ix_products_name_fulltext ON products(name)
```

Requiere MariaDB >= 10.1.4 (soportado en todas las versiones actuales).
