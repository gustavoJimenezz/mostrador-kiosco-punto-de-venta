# Ticket 4.2 — Ajuste de Escalado DPI (High DPI Scaling)

**Fecha:** 2026-03-13

## Resumen

Configuración centralizada de la política de escalado High DPI para que la UI del POS se vea nítida y bien proporcionada en el rango de monitores típicos de un kiosco: notebooks 14" con resolución 1920x1080 al 125% y monitores de escritorio 19" a 1280x1024 (ratio 4:3). La configuración debe aplicarse obligatoriamente antes de instanciar `QApplication` para que Qt la tome en cuenta desde el inicio.

## Cambios Principales

- Creado `src/infrastructure/ui/app_config.py` con la función `configure_high_dpi()`
- Aplicada política `HighDpiScaleFactorRoundingPolicy.PassThrough` para usar el factor de escala exacto del SO (sin redondeo a enteros)
- Forzado el atributo `AA_UseHighDpiPixmaps` para renderizado nítido de íconos y recursos en pantallas HiDPI
- Soporte de la variable de entorno `QT_SCALE_FACTOR` para ajuste fino en producción sin recompilar
- Modificado `src/main.py` para invocar `configure_high_dpi()` como primera acción dentro de `main()`, antes de `QApplication(sys.argv)`

## Flujo de Trabajo

- **Punto de entrada:** `main()` en `src/main.py` al iniciar la aplicación
- **Proceso:** `configure_high_dpi()` llama a `QApplication.setHighDpiScaleFactorRoundingPolicy(PassThrough)` y `QApplication.setAttribute(AA_UseHighDpiPixmaps, True)` — ambos son métodos de clase que actúan globalmente sin necesidad de una instancia
- **Ajuste fino:** Si `QT_SCALE_FACTOR` está definida como variable de entorno, Qt la aplica automáticamente; la función la detecta y la loguea en stdout para diagnóstico
- **Resultado:** `QApplication` se instancia con la política ya activa, y toda la UI hereda el comportamiento de escalado correcto

```
[main()] → [configure_high_dpi()] → [setHighDpiScaleFactorRoundingPolicy(PassThrough)]
                                   → [setAttribute(AA_UseHighDpiPixmaps)]
                                   → [log QT_SCALE_FACTOR si aplica]
         → [QApplication(sys.argv)] → [UI escalada correctamente]
```

## Archivos Afectados

| Archivo | Cambio |
|---------|--------|
| `src/infrastructure/ui/app_config.py` | Creado — encapsula toda la configuración High DPI |
| `src/main.py` | Modificado — importa y llama `configure_high_dpi()` antes de `QApplication` |

## Notas Técnicas

- En PySide6 6.x, `Qt.AA_EnableHighDpiScaling` fue eliminado (siempre activo); `AA_UseHighDpiPixmaps` está activo por defecto pero se fuerza explícitamente para compatibilidad con versiones menores anteriores.
- La política `PassThrough` es la recomendada para aplicaciones con layouts fluidos: respeta fracciones de escala como 1.25 o 1.5 sin truncarlas, evitando que botones o fuentes queden cortados.
- La política alternativa `RoundPreferFloor` puede usarse si se detectan artefactos visuales en resoluciones no estándar; es un cambio de una línea en `app_config.py`.
- La validación visual en monitores reales (1280x1024 y 1920x1080 al 125%) debe realizarse en Windows 11, ya que Linux usa un pipeline de escalado diferente al de Win32.
