# Adaptador de Importación Masiva (Polars)

**Fecha:** 2026-03-13
**Ticket:** 3.2 — Epic 3: Interfaz de Usuario y Adaptador Primario

## Resumen

Implementación del adaptador de carga masiva de listas de precios de proveedores usando **Polars** como motor de lectura CSV/Excel. Se introduce `BulkPriceImporter` (infraestructura) para parsear y validar el archivo, `UpdateBulkPrices` (use case) para el upsert atómico en MariaDB con historial de precios, e `ImportDialog` (UI) con su presenter para el flujo keyboard-first activado por F9. El lote se procesa en un `ImportWorker` (QThread) para no bloquear el hilo principal de Qt.

## Cambios Principales

- Creado use case `UpdateBulkPrices` con DTOs `ProductImportRow`, `ImportRowError` e `ImportResult`
- Upsert masivo: un único `SELECT IN` → INSERT masivo para nuevos → UPDATE individual por cambio de costo → INSERT masivo de historial → un único `commit()`
- Creado `BulkPriceImporter` con soporte de formato numérico argentino (coma decimal, punto de miles): `"1.250,50"` → `Decimal("1250.50")`
- Errores de fila no abortan el lote: se acumulan en `ImportRowError` y se reportan al final
- Columnas opcionales con defaults: `margin_percent=30`, `stock=0`, `min_stock=0`
- Añadido `ImportWorker(QThread)` a `db_worker.py`: combina errores de validación del parser con los del use case
- Creado `ImportPresenter` (Python puro, sin PySide6) con protocolo `IImportView`
- Creado `ImportDialog` (QDialog) con progress bar indeterminado y QFileDialog
- Shortcut F9 en `MainWindow` que abre `ImportDialog`
- 24 tests unitarios con datos inline (sin archivos fixture externos)

## Flujo de Trabajo

### Importación masiva (F9)

```
[Cajero presiona F9]
        │
        ▼
[MainWindow._on_open_import()]
  ImportDialog.exec()           ← modal, bloquea visualmente
        │
        ▼
[ImportDialog: botón "Seleccionar archivo…"]
  QFileDialog → Path al CSV/Excel
        │
        ▼
[ImportPresenter.on_select_file_requested()]
  view.show_status("Procesando: lista.csv...")
  view.show_progress(True)
  view.enable_select_button(False)
        │
        ▼
[ImportWorker.run()]             ← hilo separado (QThread)
        │
        ├── BulkPriceImporter.parse(file_path)
        │     pl.read_csv(path, infer_schema_length=0)   ← todo como String
        │     validación schema → ValueError si faltan columnas requeridas
        │     iter_rows(named=True) → validación fila por fila
        │     errores acumulados en ParseResult.errors
        │
        ├── UpdateBulkPrices.execute(valid_rows)
        │     SELECT barcode, id, current_cost WHERE barcode IN (...)
        │     INSERT masivo → productos nuevos
        │     UPDATE + INSERT price_history → productos con costo modificado
        │     skip → productos sin cambio de costo
        │     COMMIT único
        │
        └── result.errors.extend(parsed.errors)  ← combinar errores
                │
                ▼
        Signal import_completed(ImportResult)
                │
                ▼
[ImportPresenter.on_import_completed(result)]
  view.show_progress(False)
  view.show_status("Nuevos: 120 | Actualizados: 45 | Sin cambios: 35 | Errores: 2")
```

**Punto de entrada:** Atajo F9 en `MainWindow` o flujo posterior de botón en UI.

**Proceso:** `BulkPriceImporter` parsea y valida; `UpdateBulkPrices` ejecuta el upsert vía SQLAlchemy Core con mínimas consultas a la DB.

**Resultado:** `ImportResult` con contadores de insertados, actualizados, omitidos y lista de errores por fila.

## Notas Técnicas

### Formato numérico argentino

El algoritmo de conversión prioriza el formato local sobre el inglés estándar:

```python
# Algoritmo en BulkPriceImporter._parse_decimal()
"1.250,50"  →  remove(".")  →  "1250,50"  →  replace(",", ".")  →  "1250.50"  →  Decimal("1250.50")
"500,00"    →  remove(".")  →  "500,00"   →  replace(",", ".")  →  "500.00"   →  Decimal("500.00")
"1250"      →  sin cambios                                                     →  Decimal("1250")
```

> **Advertencia:** Si un proveedor entrega CSV con formato inglés (`"1250.50"`), el punto se trata como separador de miles y se elimina → `Decimal("125050")`. En ese caso, el proveedor debe entregar sin separador de miles (`"1250.50"` no es soportado) o usar formato argentino (`"1.250,50"` o `"1250,50"`).

### Defaults para columnas opcionales

Los valores por defecto usan enteros (sin punto decimal) para evitar la ambigüedad del separador:

```python
_OPTIONAL_DEFAULTS = {
    "margin_percent": "30",   # → Decimal("30")
    "stock": "0",
    "min_stock": "0",
}
```

### Estrategia de upsert (performance)

Un único `SELECT IN` para todos los barcodes del lote, seguido de operaciones masivas:

```
SELECT barcode, id, current_cost WHERE barcode IN (todos_los_barcodes)
         │
         ├── barcodes NO encontrados → INSERT masivo (executemany)
         │
         └── barcodes encontrados
                 ├── costo igual    → skip (skipped++)
                 └── costo distinto → UPDATE individual + acumular en price_history
                                       INSERT masivo price_history (executemany)
COMMIT único  ←  todo o nada
```

### Patrón MVP aplicado al ImportDialog

`ImportPresenter` es Python puro (no importa PySide6), testeable con `FakeImportView`:

```
ImportDialog (View)           ImportPresenter           ImportWorker (QThread)
───────────────────           ───────────────           ─────────────────────
QFileDialog → Path         →  on_select_file_req()
show_progress(True)        ←  view.show_progress()
                              retorna Path           →  worker.start()
                                                        BulkPriceImporter.parse()
                                                        UpdateBulkPrices.execute()
                                                        Signal import_completed
on_import_completed        ←  (conectado en dialog)
show_status(resumen)       ←  view.show_status()
```

### Errores de fila no abortan el lote

Un archivo de 1.000 filas con 10 filas inválidas produce `ImportResult` con 990 procesadas y `len(result.errors) == 10`. El operador ve el resumen completo al finalizar.
